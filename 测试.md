# cc-mvp 测试清单

本文档用于手动测试 `extensions/cc-mvp` 当前已实现的 MCP / HTTP bridge / 场景脚本能力。

建议测试顺序：

1. 先做环境与桥接检查
2. 再做基础场景 / 节点 / prefab 测试
3. 再做真实预览与真实构建测试
4. 最后做 AI 友好高层工具测试

## 统一 PowerShell 调用模板

如果你要直接调用 `/tool`，推荐统一使用这个模板，只替换 `$body`：

```powershell
$body = @{
  name = "scene_get_current_scene"
  arguments = @{}
} | ConvertTo-Json -Depth 20

Invoke-RestMethod `
  -Method Post `
  -Uri "http://127.0.0.1:17321/tool" `
  -ContentType "application/json" `
  -Body $body
```

如果只是查看原始文本，也可以这样：

```powershell
$body = @{
  name = "scene_get_current_scene"
  arguments = @{}
} | ConvertTo-Json -Depth 20

Invoke-WebRequest `
  -Method Post `
  -Uri "http://127.0.0.1:17321/tool" `
  -ContentType "application/json" `
  -Body $body
```

## 一、测试前准备

- Cocos Creator 3.8.7 已启动并打开项目 `E:\CocosWorkspace\Test33`
- 扩展 `cc-mvp` 已启用
- 执行过：

```bash
cd extensions/cc-mvp
npm install
npm run build
```

- 场景中准备至少一个可见节点，例如：
  - `Canvas/Player`
  - `Canvas/Enemy`
- 项目中至少存在一个场景资源，例如：
  - `db://assets/scene.scene`
  - 或 `db://assets/scenes/Main.scene`
